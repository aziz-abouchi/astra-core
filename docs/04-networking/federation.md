# Fédération dans Heaven Core

La fédération permet de connecter plusieurs clusters Heaven Core.

## Objectifs

- interopérabilité,
- isolation,
- sécurité,
- scalabilité.

## Modèle

Chaque cluster :

- possède son propre scheduler,
- possède ses propres capacités,
- peut exposer des services.

## Fédération typée

Les clusters échangent :

- des capacités fédérées,
- des protocoles fédérés,
- des identités typées.

## Routage fédéré

Le routage peut traverser plusieurs clusters :

- routage multi-domaines,
- QoS fédérée,
- supervision fédérée.

## Sécurité

- isolation stricte,
- capacités non transférables par défaut,
- validation cryptographique.
